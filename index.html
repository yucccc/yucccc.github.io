2017年度总结

回望2017，确实是坎坷但却丰收的一年

这一年，我看了这些书

  • 深入理解react和redux （完）

初衷
更多的原因是双十一打折(买了两本书) 其二是因为公司要开展新项目，我打算使用react全家桶，因此需要稍微深入学习以能够快速应付业务场景.

过程
看这本书大部分时间是在挤公交和挤地铁当中度过的，虽然看完了，但是十分的粗略(事实证明: 在车上不适合看实战类型的书籍,因为看完就忘)，在现在看来，书中的讲解某些技术都已经过时, 是怪程墨大神写书的速度呢? 还是怪react的更新速度呢 🤔目前 react 16.2 react-router 4.2.0 .
当然书还是很赞的,就像书中结语说的: 这本书介绍的具体技能很快就会过时，但是我们探讨的原理则永远都不会过时
虽然react和readux给了限制(至少从vue切换到react的我是这么觉得的)，但是正如作者所说的那样: “ 限制能够确保程序按照可控的方式进行”。

  • 富爸爸穷爸爸（完）

初衷
对于自己的理财能力，一直觉得十分欠缺，其实在工作了一段时间后，也许是受到身边事物的影响，也总在思索自己还能写多少年的代码，不写代码了能去干嘛，于是在豆瓣上找到了这本奇书以乞求能改变接下来数十载的年华。

过程
书中讲述了作者是如何从小养成理财观念，如何让钱为其工作，而不是为钱而工作，其实第一次看到这句话的时候（依然在挤地铁）内心是一阵的无力感，我们不都是为钱而工作吗？看着这一车人 早起 晚归 看似忙碌而充实的一天，眼神却无一不透露空洞，有谁能挣脱这不为钱而工作的牢笼？不都过着正如那句鸡汤所说的: 你听过许多道理，却依然过不好这一生。
也许我从书中懂得这些道理，但我又能做出什么改变？ 你（作者）有一个高材生老爸，一个成功商人干爹，简直就是开挂的一生。
终于，我还是从书中学得一些，有了更好的理财观念，并且在年底收获了一笔意想不到的财富，也许印证了那句话: 读书并不能让你立刻得到回报，但是会潜移默化的影响你的一生。

  • 深入理解ES6 (30%)

初衷
由于这边书的作者是高程的作者写的，所以一直是想看的(但是苦于价格昂贵)，同样是双十一打折(react那本只是顺带买的)，之前看过阮一峰ES6入门，相比之下这本书更具深度，遗憾的是说是ES6就真的只有ES6，ES7 8 并没涉及到，阮一峰老师的更适合当api查找，由于这本书并没有kindle版，所以纸质版的一直没怎么看(加班加班加班…)

  • 程序员跳槽全攻略 (完)

初衷
呃~~ 并没什么初衷，就是kindle做活动，免费下载，于是花了几个晚上看完了（大概只有90页？），发现了惊喜的意外收获，当然不是如何跳槽，书中更多的是教你如何规划自己职业生涯，如何选择公司或者技术栈以获取更好的职业经验，同时还普及了一波 php知识 ，遥想当年 jsp大火，作者去图书馆借不到 jsp的书籍，于是看了一波php，谁知道还因为php获得一份不错的工作,又有谁知道今时今日 jsp早已淘汰，而php依旧火热。

  • JavaScript设计模式与开发实践 (30%)

  • css揭秘（20%）

初衷
总感觉自己写着写着，css都不会了，并不能写的优美，其实我也发现这是大部分前端开发者的问题同时最容易忽略的问题，以为把页面写出了就完事，一旦需要改动，就容易牵一发动全身，遇到需要稍微复杂的动画网上cart + c + v 一顿操作。

这一年，我干了这些事

写了一个vue2 + node 仿锤子商城目前获得 450个 start 其实并没写的多好 但是还挺适合入门到进阶的
GitHub： 基于 vue+node+mongodb 实现一个电子商城

写了一个vue插件开发的教程，可做外一种思路参考.
地址: vue插件开发与发布
GitHub: 一个vue手机虚拟支付数字键盘插件

写了一个移动端1px的解决样式 主要还是自己用，也发现了一些问题，比如不支持圆角啊（需要自己给圆角）作为表格使用的时候层级问题啊…
不过也不失为一个解决方案
GitHub : 解决移动端1px问题

也写了其他一些零碎的文章，其实大部分都是自己学习之后的作品，并不能说是多么深入与严谨的产物.

2018年，我打算干这些事

  1. 深入理解ES6（完）

  2. JavaScript设计模式 (完)

  3. 在写一个新的项目 《尬聊》 使用的是 react16 全家桶 + node.js .

这也许是最后一个在GitHub上写的项目了(什么时候开始全力动工就不知道了)，因为写项目这事情虽然造福了广大同道gay人,不论是想学习这门框架抑或需要一种解决方案或者实现思路都提供了一个良好的入口，但确实是一件吃力不讨好的事情，耗费的时间也极多.

  1. 在写一个react-zoomimg 控件 用于图片的缩放
  2. 整理常用css集合，达到一定程度会放到GitHub
  3. 深入研究react技术栈

感想

这一年更多的是考虑职业的发展，关于从vue切换到react，不过都是框架，随时都可能淘汰，但是由于vue实在太优秀了，导致写多了，可能连js都忘了怎么写，只会指令了。“当有一天文档更新，指令更新，又必须去重新查阅文档 ”，这句话来自xxx一位国际友人，我觉得不无道理，而react不一样，看一次文档，基本以后都不用看了，所以在新项目中选择了react技术栈，只是为了写更多的js，为了不把最初的东西给丢了。
下一年更多的是深入研究react与自动化构建.而不是无头苍蝇，前端轮子娱乐圈，层出不尽的框架与技术或者概念，究其到底，不过JavaScript在搞鬼，因此适当了解即可，别耗费太多时间.
同时寻求一些其他的机会🤑🤑.
